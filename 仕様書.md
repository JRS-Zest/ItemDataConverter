# ItemDataConverter 仕様書

## 概要
Red Stone (JRS) の `item.dat` および `textdata.dat` を読み込み、暗号化を復号化し、表示用JSONに一括変換するC#アプリケーション。

## 目的
現在の4ステップ処理を1つのアプリケーションで完結させる：
1. ~~ItemDatDecryptorGui で item.dat を復号化~~
2. ~~複合itemdatからjson抽出.py で JSON 変換~~
3. ~~rsp_tooltip_generator.py で整形~~
4. ~~render_rsp_items.py で最終整形~~

↓

**ItemDataConverter で全て一括処理**

---

## 機能要件

### 1. 入力ファイル自動取得
レジストリからゲームインストールパスを取得し、必要なファイルパスを自動構成する。

#### レジストリキー
```
HKEY_CURRENT_USER\Software\L&K Logic Korea\Red Stone for Japan
```
- 値: `path` または `Excute Folder`

#### ファイルパス
| ファイル | 相対パス |
|---------|---------|
| item.dat | `Data\Scenario\Red Stone\item.dat` |
| textdata.dat | `Data\textData.dat` |

### 2. 復号化処理
RedStoneLib の `PacketCrypt` を使用。

#### 暗号化/復号化フロー
```csharp
// 1. ファイル先頭4バイトから rawKey を読み取り（平文）
int rawKey = BitConverter.ToInt32(data, 0);

// 2. 復号キーを生成
uint decodeKey = PacketCrypt.GenerateScenarioDecodeKey(rawKey);

// 3. 暗号化データを復号
byte[] decrypted = PacketCrypt.DecodeScenarioBuffer(encryptedData, decodeKey);
```

### 4. textdata.dat 構造

textdata.dat は**暗号化されていない**バイナリファイル。  
マーカーパターンを検索してグループを特定し、260バイトブロック単位で読み込む。

#### マーカーパターン
| グループID | マーカー（hex） | サイズ | 内容 |
|-----------|----------------|--------|------|
| 374 | `76 01 00 00 00 00 00 00` | 8B | エフェクトテンプレート |
| 187 | `00 00 BB 00 00 00 00 00 00 00` | 10B | オプションテンプレート |

#### 解析ロジック（TesxDatから抽出json.py より）
```python
# マーカー検索
search_targets = [
    b'\x76\x01\x00\x00\x00\x00\x00\x00',      # 374
    b'\x00\x00\xBB\x00\x00\x00\x00\x00\x00\x00', # 187
]

# グループID取得
if idx == 1:  # 187の場合
    group_id_val = struct.unpack('<I', content[pos+2:pos+6])[0]
else:
    group_id_val = struct.unpack('<I', content[pos:pos+4])[0]

# データ開始位置
if g_id == '187':
    current_ptr = m['offset'] + 6
else:
    current_ptr = m['offset'] + 4
```

#### ブロック構造（260バイト）
| オフセット | サイズ | 内容 |
|-----------|--------|------|
| 0-3 | 4B | data_id (int32, little-endian) |
| 4-259 | 256B | 文字列 (Shift_JIS, null終端) |

#### タグ除去
読み込んだ文字列から以下を削除：
- `<n>`
- `<c:LTYELLOW>`
- `<c:LTGREEN>`
- `<c:LTRED>`
- その他の `<c:色名>` パターン

#### 出力形式
```json
{
  "374": {
    "0": "防御力 [+0]",
    "1": "ブロック率 [+0]％",
    ...
  },
  "187": {
    "0": "追加ダメージ [+0]",
    ...
  }
}
```

## 出力形式

### 最終出力 JSON 構造
```json
{
  "0": [
    "ゴールド",
    "<基本情報>",
    "- 赤POT",
    "<要求能力値>",
    "<着用可能な職業>",
    "<DropLv/係数>",
    "- ドロップレベル 1",
    "- ドロップ係数 1000",
    "- スタック数 255"
  ],
  "1": [
    "スモールヒールポーション",
    "<基本情報>",
    "- 赤POT",
    "- HP回復 40ポイント",
    "<要求能力値>",
    "<着用可能な職業>",
    "<DropLv/係数>",
    "- ドロップレベル 1",
    "- ドロップ係数 1000",
    "- スタック数 10"
  ]
}
```

### 各セクション
1. **アイテム名**
2. **`<基本情報>`**
   - アイテムタイプ
   - 攻撃力（武器の場合: `LowAP~HighAP (速度 秒)`）
   - 射程（ある場合）
   - UniqueEffects（テンプレート置換済み）
3. **`<要求能力値>`**
   - レベル（0以外の場合）
   - 力/敏捷/健康/知恵/知識/カリスマ/幸運（0以外の場合）
4. **`<着用可能な職業>`**
   - 職業リスト（全職業の場合は性別制限のみ表示）
5. **`<DropLv/係数>`**
   - ドロップレベル
   - ドロップ係数
   - スタック数

## テンプレート置換ルール

### プレースホルダ
- `[0]`, `[1]`, `[2]`, `[3]`: そのまま数値で置換
- `[+0]`, `[+1]`: 正の場合 `+` を付加
- `[-0]`, `[-1]`: `-` を付加

### 値の範囲表示
LowValue と HighValue が異なる場合: `min~max` 形式で表示

### 無効値
- `65535` (0xFFFF): スキップ（無効なエフェクト/オプション）
- `0`: 表示しない（コンテキスト依存）

## 重要な実装方針

### 移植時の絶対ルール
1. **既存Pythonコードと完全に同一の出力を得ること**が最優先
2. 省略・解釈変更・最適化は禁止（出力結果が変わるため）
3. 既存コードのロジックを1行ずつ忠実にC#へ移植する
4. 不明点がある場合は、既存コードの動作を優先する

### 移植対象ファイル（この順序で処理を実行）
| 順序 | ファイル | 処理内容 |
|------|---------|---------|
| 1 | MainForm.cs (参考用ItemDatDecryptorGui) | item.dat 復号化 |
| 2 | TesxDatから抽出json.py | textdata.dat 解析 |
| 3 | 複合itemdatからjson抽出.py | バイナリ→JSON変換 |
| 4 | rsp_tooltip_generator.py | テンプレート置換・中間整形 |
| 5 | render_rsp_items.py | 最終整形 |

### 検証方法
- 既存Pythonスクリプトの出力と、新C#アプリの出力をJSON比較し、**完全一致**することを確認する

## 備考

### 終端マーカー
item.dat のファイル終端マーカーは `DF 02 00 00 00 00`（Scenario.dat 系）と `DA 02 00 00 00 00 65 00 00`（Data.dat 系）の2パターンがある。正式版では itemCount ベースで解析するため問題なし。
